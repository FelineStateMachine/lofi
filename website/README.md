# lofi.host

The Docusaurus site for [lofi.host](https://lofi.host): the landing page, the user docs rendered
from the repo's `docs/` folder, the API reference generated from JSDoc, and the `llms.txt` /
`llms-full.txt` corpus.

## Layout

- `docusaurus.config.ts` — reads `../docs` as the docs source; excludes repo-only content
  (`spikes/`, `devx-contract.md`, `seed.md`).
- `docs-manifest.json` — the single source of truth for which docs are published and in what order.
  `sidebars.ts` builds the sidebar from it and `../tools/site_llms.ts` builds `llms.txt` from it.
- `api-gen/` — generated by `../tools/api_docs.ts` from `deno doc --json`; gitignored.
- `static/llms.txt`, `static/llms-full.txt` — generated by `../tools/site_llms.ts`; gitignored.
- `src/pages/index.tsx` + `src/css/landing.css` — the landing page, ported from the standalone
  lofi-site.

## Develop

From the repository root:

```sh
deno task site:dev     # generate API pages + llms files, then start the dev server
deno task site:build   # full production build into website/build
```

Docs content edits belong in `../docs/*.md` (plain CommonMark, no frontmatter — the same files
render on GitHub and JSR). New docs must be added to `docs-manifest.json` to appear in the sidebar
and llms.txt.

## Deploy

`.github/workflows/site.yml` builds on every PR touching site inputs and deploys to Cloudflare
Pages (project `lofi-host`, domain lofi.host) on every push to `main`.
